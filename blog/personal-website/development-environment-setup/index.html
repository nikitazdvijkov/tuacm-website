How to Set up Your Webdev Environment

Setting up the tools to make changes to the website.
Posted 19 August 2020 at 11:42 PM
By Nikita Zdvijkov

Just as in the last tutorial, we won't go into great depth about why we're doing what we're doing. We're just trying to set everything up for now and fill in the details later.

Make sure to take good notes of what you do and what tutorials you follow for the git setup so we can put them in this article later on. Both the author and the editor set git up a long time ago, so we don't remember what works best for you.

Mac

Follow this tutorial to set your Mac up for web development. You don't have to install NodeJS &mdash you can always do it later — but it won't hurt. Use VSCode for a text editor for now. You might want to check out other text editors later, but it won't matter for this tutorial.

Once you've finished the tutorial, open up your terminal and type the following commands: 

computer:~ user$ git config --global user.name "John Doe"
computer:~ user$ git config --global user.email "johndoe@example.com"

Windows

First, set up the Windows Subsystem for Linux. Then, install VSCode. Lastly, run the following commands in your terminal. 
user@computer:~/dev$ git config --global user.name "John Doe"
user@computer:~/dev$ git config --global user.email "johndoe@example.com"

Linux

Install VSCode, then run the following commands in your terminal. 

user@computer:~/dev$ git config --global user.name "John Doe"
user@computer:~/dev$ git config --global user.email "johndoe@example.com"

Connect GitHub to Netlify

  1. Go to GitHub. Use the website to create a new Git repository. Call it whatever you want as long as it doesn't have spaces in the name. It doesn't matter whether it's a public or private repo. Don't bother initializing with README.md or .gitignore.
  2. At this point, GitHub will show you instructions for getting started using your repo. Follow the first set of instructions (involves cloning the repo).
  3. Put your index.html file without the folder containing it into your cloned git repo. At this point, your repo should only have one non-hidden file. Type the command ls in the terminal to list all non-hidden files in the current directory. Type the command ls -a to list all files.
  4. Go to Netlify and follow the prompts to connect your git repo to Netlify.

Use git

Research what these commands do. As long as you're not collaborating or doing anything fancy, these should be all you need:

  • git clone
  • git add -A
  • git commit -m "Changed this thing"
  • git status
  • git log
  • git push origin master
