This file is not currently on the internet — it's still sitting on your computer. We still have to buy a domain and put it up on a server.

Buying a Domain

Only this step will cost you money, but you can find cheap domain names for less than three bucks a year. For example, the editor's website only cost a little over a dollar.

From what I've read, most pro webdevs use NameCheap to register names. I've used it without any problems, so I'll recommend it. Here's my affiliate link to NameCheap — meaning I get a kickback. Create Accounts

Sign up for Github and Netlify. I recommend signing up for Netlify using your GitHub account instead of email or anything else.

Deploy to Netlify Without Custom Domain

Before we can deploy to the custom domain, we have to drag and drop the my-first-website folder into Netlify.

At this point, Netlify will automatically give you a link of the form https://example.netlify.app where your html document is published.

Connect to Custom Domain

At this point, we have to go through a bunch of menus, so follow the screenshots.

Enter the domain you just bought, click verify, then click yes.

Skip adding a new record for now. If you want, you can add it later.

Copy the nameservers Netlify gives you and paste them into NameCheap. NameCheap might give you different nameservers that it gave me. The nameservers should be the same in Netlify and NameCheap — it's a mistake that they're different in the screenshots.

You'll have to switch from NameCheap Basic DNS to Custom DNS. Make sure to hit the green check mark to save.

Go back to Netlify and check to see if it worked. It can take up to a day for these changes to take effect, so don't worry if it doesn't happen immediately.

After this, visitors to your custom domain will see your index.html file. Note that they don't have to go to example.com/index.html — just the domain name by itself will work.
